# Telegram Status Board Usage Notes

## Best fit
- direct-message workflows
- mobile-first control view
- one-message operational summaries
- quick health checks from Telegram

## Not the goal
- replacing a full browser dashboard
- showing every cron, every session, every log line
- rendering heavy charts inside Telegram

## Recommended interaction model
1. Show a compact board first
2. Offer drill-down on request
3. Refresh the same structure consistently

## Good follow-up questions
- show sessions
- show subagents
- show cron failures
- show cost breakdown
- refresh board
